# 添加技能

技能可以增强智能体的工作能力。应用负责“连接外部系统”，技能更像是“让智能体学会某种做事方式或工具能力”。

简单理解：技能是智能体的“工作方法包”。它不一定连接外部系统，而是告诉智能体在某类任务里应该怎么判断、怎么调用工具、按什么格式输出。

## 1. 技能类型

常见技能包括：

- **指南类技能**：告诉智能体按什么规范、流程或角色来工作。
- **工具类技能**：让智能体可以调用浏览器、搜索、代码仓库、数据库、WASM 工具等。
- **自定义技能**：你为某个智能体单独导入或编辑的技能。

简单理解：技能负责“怎么做”。

## 技能和 Persona 模板的关系

Persona 模板会推荐一组技能，但不会限制你只能用这些技能。模板负责给智能体一个适合的角色起点，技能负责补充具体能力。

例如一个“合同审查助手”模板可能默认推荐“合同风险识别”“条款摘要”“法务审查格式”几个技能。你创建后可以保留默认技能，也可以关闭不需要的技能，再添加自己的自定义技能。

## 2. 创建智能体时添加技能

在创建向导最后一步切换到 `Skills`。

你可以：

- 查看模板推荐技能。
- 使用筛选项查看工具类或指南类技能。
- 打开需要的技能。
- 提交创建。

## 3. 创建后添加技能

智能体创建完成后：

1. 打开智能体详情页。
2. 点击顶部 `Skills` 标签。
3. 搜索或筛选技能。
4. 打开技能开关。
5. 如需配置，点击 `Configure`。

## 4. 导入自定义技能

在 `Skills` 页面中，可以导入自定义技能。

常见方式：

- `Import from URL`：从 URL、Registry、GitHub、ClawHub、Skills.sh 或压缩包地址导入。
- `Upload Skill`：上传 `.zip` 技能包。
- 在线编辑：编辑已导入的智能体级技能。

### 从 URL 导入

1. 点击 `Import from URL`。
2. 搜索技能，或直接粘贴 URL。
3. 点击预览。
4. 如果有多个技能，选择要导入的条目。
5. 点击导入。

1. 点击 `Upload Skill`。
2. 选择 `.zip` 文件。
3. 等待上传完成。
4. 回到技能列表启用该技能。

## 5. 配置技能数据声明

部分技能需要声明它会处理的数据类型。

常见数据标签：

- `data.public`：公开数据。
- `data.internal`：内部数据。
- `data.commercial`：商业数据。
- `data.pii`：个人敏感信息。
- `data.confidential`：机密数据。
- `data.secret`：秘密数据。

如果技能涉及个人信息、机密资料或敏感业务数据，建议联系管理员确认是否需要使用本地模型、开启审计或限制访问。

## 新手建议

- 不要一次添加太多技能。
- 先添加和当前任务最相关的 1-3 个技能。
- 工具类技能可能依赖智能体运行时，若页面提示不可用，请换用支持的智能体类型。
- 技能启用后，先用简单任务测试是否生效。
