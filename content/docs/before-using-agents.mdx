# 使用智能体前你需要知道的事

在 Zentrix 中，智能体不是一个只能聊天的空白机器人，而是可以在企业空间里被装配出来的 AI 工作伙伴。

你可以把创建智能体理解为一次“企业级装配”：管理员先为企业空间准备好可用的 Persona 模板、技能、应用、模型服务和 Channel 连接方式，用户再根据自己的任务选择合适的组件，组装出适合具体业务场景的智能体。

## 1. 什么是企业空间里的智能体装配

在企业使用场景中，一个智能体通常由这些部分组成：

| 组成部分 | 它决定什么 | 简单理解 |
| --- | --- | --- |
| Persona 模板 | 智能体适合做什么、像什么职业角色、默认回答风格 | 角色定位 |
| 技能 Skill | 智能体在专业任务里怎么做、按什么流程做、能调用哪些工具 | 专业能力 |
| 应用 App | 智能体能连接哪些系统、读取哪些业务资料 | 数据和系统连接 |
| Channel | 用户可以从哪里和智能体对话，例如网页、钉钉、企业微信、飞书 | 对话入口 |
| 模型服务 | 智能体调用哪个 AI 模型来理解和回答 | 大脑 |

这些能力通常由管理员先配置好。普通用户创建智能体时，不需要从零理解所有底层设置，只需要在企业空间提供的范围内选择适合自己的组合。

## 2. Persona 决定智能体的职业角色

创建智能体时，Persona 模板是很重要的起点。

选择 Persona 时，不需要过度纠结。选择你认为和自己任务最相关的 Persona 模板即可。它只是一个通用的初级模板，帮助智能体先获得一个职业角色和基础能力，不会决定智能体的最终技能。真正适合你的智能体，需要继续通过添加不同的 Skill 和 App 来打造。

Persona 决定了：

- 这个智能体适合做哪些任务。
- 它偏向什么职业角色。
- 它应该如何理解用户请求。
- 它回答问题时应保持什么风格。
- 它默认会预装哪些通用技能。

例如：

| Persona 类型 | 更适合的任务 |
| --- | --- |
| 职场效率助手 | 邮件、纪要、周报、翻译、日常办公 |
| 市场情报助手 | 行业情报、竞品追踪、趋势研判 |
| 内容营销助手 | 内容规划、SEO 优化、多渠道分发 |
| 工程师助手 | 代码修复、PR Review、故障诊断 |
| 财务助手 | 发票、合同、费用、财报处理 |
| HR 助手 | 招聘、入职、FAQ、培训文档 |

简单理解：Persona 决定“这个智能体像谁、适合做什么”。

## 3. Persona 会预装通用技能，但不等于最终能力

Persona 模板通常会带上一些角色通用技能。

例如，一个“职场效率助手”可能会预装：

- 总结文档。
- 整理会议纪要。
- 生成周报。
- 翻译内容。

这些技能能帮你快速开始，但它们只是基础能力。

如果你的任务更专业，比如合同审查、销售线索分析、代码审查、知识库问答，就需要继续添加更具体的 Skill。

## 4. Skill 用来补齐专业任务能力

Skill 决定智能体在某类任务中“怎么做”。

Persona 更偏向角色定位，Skill 更偏向专业执行方法。

举例：

| 目标任务 | Persona 可能选择 | 还可以添加的 Skill |
| --- | --- | --- |
| 生成销售跟进建议 | 销售助手 | 客户分层、线索评分、跟进话术 |
| 审查合同风险 | 法务助手 | 合同风险识别、条款摘要、审查清单 |
| 做代码审查 | 工程师助手 | PR Review、代码规范检查、故障诊断 |
| 总结企业文档 | 知识助手 | 文档检索、引用总结、知识库问答 |
| 生成运营报告 | 数据分析助手 | 指标解释、图表分析、报告模板 |

简单理解：Persona 让智能体“像某个角色”，Skill 让智能体“会做某件专业的事”。

## 5. App 让智能体连接企业系统

如果智能体只靠聊天输入，它只能处理你发给它的信息。

连接 App 后，智能体可以在授权范围内访问企业系统或业务资料，例如：

- 钉钉文档。
- 飞书文档。
- 企业知识库。
- CRM。
- 数据库。
- 代码仓库。
- MCP 服务。

App 解决的是“智能体连到哪里、能读取或操作什么系统”的问题。

例如，你想让智能体总结钉钉文档，就需要连接钉钉文档应用；你想让智能体辅助代码审查，就可能需要连接代码仓库或相关 MCP 服务。

## 6. Channel 决定从哪里使用智能体

Channel 是智能体的对话入口。

常见 Channel 包括：

- Zentrix 网页 Chat。
- 钉钉。
- 企业微信。
- 飞书。
- 微信。

如果只在网页里使用智能体，可以先不配置额外 Channel。

如果你希望团队成员直接在 IM 群里和智能体对话，就需要配置对应 Channel。

## 7. 创建智能体时建议按这个顺序思考

创建前先问自己 5 个问题：

1. **它要扮演什么角色？**  
   先选择合适的 Persona。

2. **它要完成什么专业任务？**  
   再选择需要的 Skill。

3. **它需要读取哪些企业资料？**  
   如果需要外部资料，就连接 App。

4. **用户从哪里和它对话？**  
   网页够用就用 Chat，需要 IM 就配置 Channel。

5. **它应该使用哪个模型？**  
   根据任务复杂度选择模型服务。

如果你是第一次创建，建议先从 Persona 模板开始，只添加最必要的 App 和 Skill。跑通一个简单任务后，再逐步增强能力。

## 8. 一个完整例子

假设你想创建一个“销售线索分析助手”。

可以这样装配：

| 装配项 | 选择示例 |
| --- | --- |
| Persona | 销售 / 客户跟进助手 |
| Skill | 线索评分、客户分层、跟进建议生成 |
| App | CRM、客户表格、销售文档 |
| Channel | Zentrix Chat，或钉钉销售群 |
| 模型服务 | 管理员提供的通用或高阶模型服务 |

这样创建出来的智能体就不只是“能聊天”，而是更接近一个面向销售场景的企业助手。

## 9. 新手最容易混淆的点

### Persona 和 Skill 有什么区别

Persona 决定角色，Skill 决定能力。

Persona 像“岗位说明”，Skill 像“专业工具和工作方法”。

### App 和 Channel 有什么区别

App 连接系统，Channel 连接人。

App 让智能体访问资料或工具；Channel 让用户从不同入口和智能体对话。

### 管理员和普通用户分别做什么

管理员通常负责：

- 配置模型服务。
- 准备可用应用。
- 提供技能库。
- 管理 Persona 模板。
- 配置企业级 Channel。
- 控制工作空间权限和安全策略。

普通用户通常负责：

- 选择 Persona。
- 按任务添加 Skill。
- 选择可用 App。
- 创建并测试智能体。
- 根据效果继续调整。
