# Zentrix 单机生产版安装手册

这份手册面向第一次安装 Zentrix 的用户。你只需要准备一台 Linux 服务器和一个交付安装包，按步骤执行即可。

本文参考离线交付包安装流程整理，适用于：

- 安装在一台服务器上。
- 使用镜像制品包安装。
- 客户现场可以没有外网。

## 1. 先理解安装包是什么

交付时通常会给你两个文件：

```text
lobsterpool-offline-amd64-版本号.tar.gz
lobsterpool-offline-amd64-版本号.tar.gz.sha256
```

第一个是安装包，第二个是校验文件，用来确认安装包没有损坏。

安装包解压后大致包含：

```text
.
├── images/                 # Docker 镜像文件，安装时会自动导入
├── runtime/                # 运行时配置、前端页面、Caddy 配置等
├── docker-compose.yml      # 生产部署用的 Docker Compose 文件
├── install.sh              # 安装、升级、回滚脚本
├── install.conf.example    # 安装配置模板
├── manifest.json           # 版本、镜像清单、校验和
└── README.md
```

你通常只需要关心这三个文件：

| 文件 | 用途 |
| --- | --- |
| `install.conf.example` | 配置模板，复制成 `install.conf` 后填写 |
| `install.sh` | 执行安装、升级、回滚 |
| `manifest.json` | 查看安装包版本和镜像信息 |

## 2. 安装前准备

### 2.1 服务器要求

建议使用一台干净的 Linux 服务器。

| 项目 | 最低要求 | 建议 |
| --- | --- | --- |
| 系统 | Linux x86_64 / amd64 | Ubuntu 22.04/24.04、Debian 12、RHEL 9 |
| CPU | 4 核 | 8 核以上 |
| 内存 | 8 GB | 16 GB 以上 |
| 磁盘 | 10 GB 可用 | 100 GB 以上，生产建议做快照 |
| Docker | 20.10+ | 越新越好 |
| Docker Compose | V2 | 命令必须是 `docker compose` |

### 2.2 必须空闲的端口

安装前请确保这些端口没有被其他程序占用：

```text
80, 443, 3000, 5432, 6379, 8443, 9000
```

检查命令：

```bash
ss -tlnp | grep -E ':(80|443|3000|5432|6379|8443|9000)\b' || true
```

没有输出通常表示这些端口没有被占用。

安装器还会使用一些观测端口，例如 `4317`、`4318`、`8888`、`9090`、`3001`。这些端口如果被占用，安装器会自动找下一个空闲端口，不需要你手动处理。

### 2.3 检查 Docker 是否可用

执行：

```bash
docker version
docker compose version
docker ps
```

如果 `docker ps` 提示权限不足，试试：

```bash
sudo docker ps
```

后续安装命令都使用 `sudo` 执行。

## 3. 上传安装包

在你的电脑上上传安装包：

```bash
scp lobsterpool-offline-amd64-版本号.tar.gz* 用户名@服务器IP:/tmp/
```

示例：

```bash
scp lobsterpool-offline-amd64-1.0.0.tar.gz* admin@192.168.1.10:/tmp/
```

登录服务器：

```bash
ssh 用户名@服务器IP
```

示例：

```bash
ssh admin@192.168.1.10
```

## 4. 校验安装包

进入 `/tmp`：

```bash
cd /tmp
```

执行校验：

```bash
sha256sum -c lobsterpool-offline-amd64-版本号.tar.gz.sha256
```

示例：

```bash
sha256sum -c lobsterpool-offline-amd64-1.0.0.tar.gz.sha256
```

看到 `OK` 就表示安装包没有损坏：

```text
lobsterpool-offline-amd64-1.0.0.tar.gz: OK
```

## 5. 解压安装包

创建临时安装目录：

```bash
sudo rm -rf /opt/lobsterpool-stage
sudo mkdir -p /opt/lobsterpool-stage
```

解压：

```bash
sudo tar xzf /tmp/lobsterpool-offline-amd64-版本号.tar.gz -C /opt/lobsterpool-stage
```

示例：

```bash
sudo tar xzf /tmp/lobsterpool-offline-amd64-1.0.0.tar.gz -C /opt/lobsterpool-stage
```

进入目录：

```bash
cd /opt/lobsterpool-stage
```

确认文件完整：

```bash
ls
```

你应该能看到：

```text
docker-compose.yml
images
install.conf.example
install.sh
manifest.json
runtime
```

## 6. 填写安装配置

复制配置模板：

```bash
sudo cp install.conf.example install.conf
```

编辑配置：

```bash
sudo vi install.conf
```

如果你不熟悉 `vi`，可以使用：

```bash
sudo nano install.conf
```

### 6.1 必填配置

找到并填写：

```bash
PUBLIC_URL=
DB_PASS=
```

示例一：有域名和 HTTPS：

```bash
PUBLIC_URL=https://ai.example.com
DB_PASS=请换成一个很强的数据库密码
```

示例二：内网试用：

```bash
PUBLIC_URL=http://192.168.1.10
DB_PASS=请换成一个很强的数据库密码
```

说明：

- `PUBLIC_URL` 是用户打开 Zentrix 的地址，不要带最后的 `/`。
- `DB_PASS` 是数据库密码，请不要使用 `123456`、`password` 这类简单密码。

### 6.2 推荐填写的路径

默认可以不改：

```bash
INSTALL_DIR=/opt/lobsterpool
BACKUP_DIR=/opt/lobsterpool/backups
CONTEXT_HOST_PATH=/var/lib/lobsterpool/contexts
```

说明：

| 配置 | 说明 |
| --- | --- |
| `INSTALL_DIR` | Zentrix 正式安装目录 |
| `BACKUP_DIR` | 升级前自动备份目录 |
| `CONTEXT_HOST_PATH` | 智能体工作空间目录 |

### 6.3 推荐填写模型配置

如果安装完成后要直接使用 AI 功能，建议填写：

```bash
FACTORY_PROVIDER_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
FACTORY_PROVIDER_KEY=你的模型服务商APIKey
FACTORY_PROVIDER_MODEL=qwen3.5-plus
```

如果你使用其他 OpenAI 兼容模型服务，把 URL 和模型名改成对应值。

### 6.4 可以留空自动生成的密钥

这些可以先不填：

```bash
JWT_SECRET=
LOBSTERPOOL_MASTER_KEY=
SPICEDB_TOKEN=
RUSTFS_SECRET_KEY=
PLATFORM_ADMIN_KEY=
```

安装脚本会自动生成，并写入：

```text
/opt/lobsterpool/.env
```

请务必备份 `/opt/lobsterpool/.env`，尤其是 `LOBSTERPOOL_MASTER_KEY`。这个密钥丢失后，平台里已经加密保存的密钥和凭证无法恢复。

### 6.5 如果服务器需要代理上网

如果服务器或容器必须通过代理访问模型服务商，可以填写：

```bash
HTTP_PROXY=http://代理地址:端口
HTTPS_PROXY=http://代理地址:端口
EGRESS_UPSTREAM_PROXY=http://代理地址:端口
```

没有代理要求就不用填。

## 7. 开始安装

确认当前目录：

```bash
pwd
```

应该输出：

```text
/opt/lobsterpool-stage
```

执行安装：

```bash
sudo bash install.sh
```

第一次安装可能需要几分钟。

安装脚本会自动做这些事：

1. 检查 Docker、Compose、磁盘和端口。
2. 读取 `install.conf`。
3. 生成生产 `.env`。
4. 逐个导入 `images/*.tar` 镜像。
5. 复制运行文件到 `/opt/lobsterpool`。
6. 启动基础服务。
7. 启动 Zentrix 全部服务。
8. 执行数据库初始化。
9. 检查一次性初始化容器是否成功。
10. 检查 `/health`、`/api/v1/version` 和前端页面。

看到类似下面的提示表示安装完成：

```text
Deployment complete
Access LobsterPool at: http://192.168.1.10
```

## 8. 检查是否安装成功

进入正式安装目录：

```bash
cd /opt/lobsterpool
```

查看服务状态：

```bash
sudo docker compose ps
```

检查 API：

```bash
curl -s http://localhost:3000/health
```

正常结果：

```json
{"status":"ok"}
```

检查版本：

```bash
curl -s http://localhost:3000/api/v1/version
```

检查网页：

```bash
curl -I http://localhost
```

如果你配置了 HTTPS 域名：

```bash
curl -I https://你的域名
```

也可以检查安装状态文件：

```bash
python3 -m json.tool /opt/lobsterpool/install-state.json
python3 -m json.tool /opt/lobsterpool/bootstrap-status.json
```

如果 `install-state.json` 里显示 `completed`，说明安装流程完成。

## 9. 浏览器初始化

用浏览器打开你在 `PUBLIC_URL` 填写的地址。

例如：

```text
http://192.168.1.10
```

或：

```text
https://ai.example.com
```

第一次打开时，按照页面提示创建第一个管理员账号。

创建完成后建议做一次测试：

1. 登录控制台。
2. 检查模型服务是否可用。
3. 创建一个测试智能体。
4. 启动智能体。
5. 发一条测试消息。

智能体能正常回复，说明平台和模型调用基本正常。

## 10. 安装后目录说明

正式安装目录默认是：

```text
/opt/lobsterpool
```

常见文件：

| 文件或目录 | 说明 |
| --- | --- |
| `/opt/lobsterpool/.env` | 生产配置和密钥，必须保护和备份 |
| `/opt/lobsterpool/docker-compose.yml` | 服务启动配置 |
| `/opt/lobsterpool/manifest.json` | 当前安装版本、镜像清单和校验信息 |
| `/opt/lobsterpool/install-state.json` | 最近一次安装或升级状态 |
| `/opt/lobsterpool/bootstrap-status.json` | 初始化容器状态 |
| `/opt/lobsterpool/backups/` | 升级前自动备份 |
| `/var/lib/lobsterpool/contexts` | 智能体工作空间目录 |

最重要的是：

```text
/opt/lobsterpool/.env
```

请不要随意删除，也不要发给无关人员。

## 11. 常用管理命令

进入安装目录：

```bash
cd /opt/lobsterpool
```

查看服务状态：

```bash
sudo docker compose ps
```

查看全部日志：

```bash
sudo docker compose logs -f
```

查看主服务日志：

```bash
sudo docker compose logs -f lobsterpool
```

重启主服务：

```bash
sudo docker compose restart lobsterpool
```

重启全部服务：

```bash
sudo docker compose restart
```

停止全部服务：

```bash
sudo docker compose down
```

启动全部服务：

```bash
sudo docker compose up -d
```

查看当前版本：

```bash
python3 -m json.tool /opt/lobsterpool/manifest.json
```

## 12. 升级

升级时，你会拿到新的安装包：

```text
lobsterpool-offline-amd64-新版本.tar.gz
lobsterpool-offline-amd64-新版本.tar.gz.sha256
```

上传、校验、解压：

```bash
scp lobsterpool-offline-amd64-新版本.tar.gz* 用户名@服务器IP:/tmp/
ssh 用户名@服务器IP

cd /tmp
sha256sum -c lobsterpool-offline-amd64-新版本.tar.gz.sha256

sudo rm -rf /opt/lobsterpool-stage-new
sudo mkdir -p /opt/lobsterpool-stage-new
sudo tar xzf /tmp/lobsterpool-offline-amd64-新版本.tar.gz -C /opt/lobsterpool-stage-new
cd /opt/lobsterpool-stage-new
```

准备配置：

```bash
sudo cp install.conf.example install.conf
sudo vi install.conf
```

请确保这些配置和旧环境一致：

```bash
PUBLIC_URL=原来的访问地址
DB_PASS=原来的数据库密码
INSTALL_DIR=/opt/lobsterpool
BACKUP_DIR=/opt/lobsterpool/backups
```

执行升级：

```bash
sudo bash install.sh
```

升级脚本会自动：

1. 读取旧版本。
2. 备份数据库、配置和旧镜像。
3. 停止旧服务。
4. 导入新镜像。
5. 保留原来的 `.env` 密钥。
6. 启动新版本。
7. 做健康检查。

升级后检查：

```bash
cd /opt/lobsterpool
sudo docker compose ps
curl -s http://localhost:3000/health
curl -s http://localhost:3000/api/v1/version
```

## 13. 回滚

如果升级后发现问题，可以回滚到升级前的版本。

进入新版安装包目录：

```bash
cd /opt/lobsterpool-stage-new
```

执行回滚：

```bash
sudo bash install.sh rollback
```

回滚会从最近一次备份恢复：

- `.env`
- `docker-compose.yml`
- `manifest.json`
- 旧版本镜像
- 主数据库
- 权限数据库

回滚后检查：

```bash
cd /opt/lobsterpool
sudo docker compose ps
curl -s http://localhost:3000/health
curl -s http://localhost:3000/api/v1/version
```

备份目录在：

```text
/opt/lobsterpool/backups/
```

## 14. 常见问题

### 14.1 安装提示端口被占用

现象：

```text
Ports already in use
```

检查占用：

```bash
ss -tlnp | grep -E ':(80|443|3000|5432|6379|8443|9000)\b'
```

处理方式：

- 停掉占用端口的旧服务。
- 或换一台干净服务器安装。

### 14.2 健康检查没有返回 ok

检查服务：

```bash
cd /opt/lobsterpool
sudo docker compose ps
```

查看日志：

```bash
sudo docker compose logs lobsterpool
sudo docker compose logs postgres
sudo docker compose logs lobsterpool-migrate
```

也可以查看安装状态：

```bash
python3 -m json.tool /opt/lobsterpool/install-state.json
python3 -m json.tool /opt/lobsterpool/bootstrap-status.json
```

### 14.3 页面打不开

先在服务器本机检查：

```bash
curl -I http://localhost
curl -s http://localhost:3000/health
```

如果服务器本机正常，但外部打不开，通常是：

- 服务器防火墙没有放行 `80/443`。
- 云服务器安全组没有放行 `80/443`。
- 域名没有正确解析到服务器 IP。
- `PUBLIC_URL` 填错。

### 14.4 智能体启动失败

查看主服务日志：

```bash
cd /opt/lobsterpool
sudo docker compose logs lobsterpool
```

检查运行时镜像：

```bash
sudo docker images | grep -E 'openclaw|ironclaw|opencode'
```

### 14.5 模型调用失败

检查配置：

```bash
cd /opt/lobsterpool
grep -E 'FACTORY_PROVIDER|OPENAI|ANTHROPIC|DASHSCOPE|HTTP_PROXY|EGRESS' .env
```

查看日志：

```bash
sudo docker compose logs tensorzero
sudo docker compose logs lobsterpool
```

常见原因：

- API Key 填错。
- 模型名称填错。
- 服务器无法访问模型服务商。
- 需要代理但没有配置代理。
- 模型服务商账号欠费、限流或无权限。

## 15. 安装完成后的检查清单

安装完成后，请逐项确认：

- [ ] 浏览器能打开 `PUBLIC_URL`。
- [ ] 已创建第一个管理员账号。
- [ ] `curl -s http://localhost:3000/health` 返回 `{"status":"ok"}`。
- [ ] `sudo docker compose ps` 没有明显失败的服务。
- [ ] `/opt/lobsterpool/.env` 已备份并限制访问权限。
- [ ] `LOBSTERPOOL_MASTER_KEY` 已安全备份。
- [ ] 已确认模型 provider 配置可用。
- [ ] 已创建并启动一个测试智能体。
- [ ] 如果是公网使用，服务器防火墙或安全组已放行 `80/443`。
- [ ] 已规划数据库和对象存储的数据备份。
