# 模型服务商管理

模型服务商决定 Zentrix 可以调用哪些 AI 模型。管理员需要先配置模型服务商和模型服务，普通用户才能在创建智能体或聊天时选择模型。

如果你只是普通使用者，看到“没有可用模型服务”时，可以把这一页发给管理员处理。

## 1. 什么是模型服务商

模型服务商就是提供模型 API 的平台，例如：

- OpenAI-compatible 服务。
- DashScope。
- DeepSeek。
- 企业内部模型网关。
- 其他兼容 OpenAI 接口的模型平台。

配置模型服务商时，通常需要准备：

| 信息 | 说明 |
| --- | --- |
| Provider URL | 模型服务的接口地址 |
| API Key | 调用模型所需密钥 |
| 模型名称 | 例如 `qwen3.5-plus`、`gpt-4.1`、`deepseek-chat` |
| 可用范围 | 哪些工作空间或用户可以使用 |

## 2. 进入管理页面

管理员登录 Zentrix 后，进入管理后台：

![后台管理概览](/images/docs/admin-overview.jpg)


```text
/admin/model-providers
```

如果页面没有权限，说明当前账号不是管理员，需要联系系统管理员授权。

![模型服务商管理页面](/images/docs/ai-provider.jpg)

## 3. 新增模型服务商

在模型服务商页面点击新增或创建，然后填写：

1. 服务商名称，例如“DashScope 生产账号”。
2. Provider 类型，通常选择 OpenAI-compatible 或对应平台。
3. API 地址。
4. API Key。
5. 保存。

填写完成后，建议立即测试连接。测试失败时，优先检查 API Key、接口地址、服务器网络和代理配置。

## 4. 添加或同步模型

模型服务商保存后，需要确认模型列表可用。

常见做法：

- 如果系统支持自动同步，点击同步模型。
- 如果需要手动填写，添加模型名称。
- 设置默认模型或主模型。
- 启用需要开放给用户的模型。

模型名称必须和服务商实际支持的名称一致。名称写错时，智能体创建成功后也可能无法正常回复。

## 5. 创建模型服务

模型服务是普通用户在创建智能体时看到的选项。

进入：

```text
/admin/model-services
```

创建模型服务时，建议填写：

- 模型服务名称，例如“通用问答模型”。
- 关联的模型服务商。
- 可用模型。
- 默认模型。
- 可使用的工作空间。
- 启用状态。

保存后，普通用户就可以在创建智能体时选择这个模型服务。

## 6. 验证是否配置成功

验证方式：

1. 切换到普通用户或目标工作空间。
2. 打开创建智能体页面。
3. 查看模型服务下拉框。
4. 选择刚创建的模型服务。
5. 创建一个测试智能体。
6. 发送一条测试消息。

如果智能体能正常回复，说明模型服务商和模型服务配置成功。

## 7. 常见问题

### 创建智能体时没有模型可选

可能原因：

- 没有创建模型服务。
- 模型服务没有启用。
- 模型服务没有授权给当前工作空间。
- 模型服务商的 API Key 无效。
- 模型列表为空。

### 测试连接失败

优先检查：

- Provider URL 是否正确。
- API Key 是否正确。
- 服务器是否能访问模型服务商。
- 是否需要配置代理。
- 模型服务商账号是否欠费或被限流。

### 智能体创建成功但聊天失败

优先检查：

- 模型名称是否写错。
- 模型服务商是否支持该模型。
- API Key 是否有调用该模型的权限。
- 是否达到模型服务商限额。
- Zentrix 服务器是否能访问外部模型接口。
