# Persona 模板选择

Persona 模板是智能体的“出厂角色配置”，负责定义智能体是谁、适合做什么、回答时应该保持什么风格。

如果你还不确定要怎么填写智能体说明、怎么搭配技能和应用，可以先从 Persona 模板开始。模板不会限制后续修改，创建完成后仍然可以调整名称、描述、模型、应用和技能。

选择 Persona 时，不需要追求一步到位。选择你认为和自己最相关的 Persona 模板即可。它只是一个通用的初级模板，不会决定智能体的最终技能。后续你需要通过添加不同的 Skill 和 App，继续打造真正适合自己业务目标的智能体。

## 什么时候需要选择模板

这些情况建议优先选择 Persona 模板：

- 你是第一次创建智能体，不知道每个配置项应该怎么填。
- 你要做的是常见场景，例如客服、销售、文档问答、合同审查、运营分析。
- 你希望系统先帮你准备好角色说明、回答风格和推荐能力。
- 你想快速验证一个想法，先跑通对话，再慢慢调整。

如果你已经非常清楚智能体要做什么，也知道要连接哪些应用和技能，可以选择空白智能体。

## 常见模板怎么选

| 你想做的事 | 建议选择的模板类型 | 适合的使用方式 |
| --- | --- | --- |
| 回复客户问题、整理常见问答 | 客服 / 问答助手 | 连接知识库或文档应用，让智能体根据资料回答 |
| 跟进客户、分析线索、生成销售建议 | 销售 / 客户跟进助手 | 搭配 CRM、表格、文档类应用使用 |
| 查询企业文档、总结文档内容 | 文档问答 / 知识助手 | 连接钉钉文档、飞书文档、企业知识库等应用 |
| 审查合同、提取风险点 | 法务 / 合同审查助手 | 添加合同审查、条款摘要等技能 |
| 生成周报、日报、会议纪要 | 办公写作 / 报告助手 | 搭配文档应用和写作类技能 |
| 分析运营数据、整理业务结论 | 数据分析 / 运营助手 | 连接表格、数据库或业务系统 |
| 协助代码理解、代码审查 | 研发 / 代码助手 | 连接代码仓库，添加代码审查或研发规范技能 |

## 选择模板后还需要检查什么

选择模板后，建议重点检查这几项：

- **名称**：是否能让团队成员一眼看懂用途。
- **描述**：是否说明了输入什么、输出什么。
- **模型**：是否适合当前任务，复杂任务可以选择能力更强的模型。
- **应用**：是否连接了智能体需要读取的数据来源。
- **技能**：是否只保留当前任务真正需要的能力。

模板只是起点，不是最终配置。你可以先用模板创建一个可用版本，再根据实际对话效果微调。

## 和创建智能体的关系

在“创建智能体”流程里，Persona 模板出现在选择创建起点的阶段。

简单记住一句话：

> Persona 模板决定“这个智能体像谁、适合做什么”；模型决定“它用什么大脑回答”；应用和技能决定“它能连到哪里、会怎么做事”。
